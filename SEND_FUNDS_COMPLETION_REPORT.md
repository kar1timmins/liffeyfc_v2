## 🔍 Wallet Type Clarification

### Liffey Account Wallet
- **Generated by**: Backend (auto-created at registration)
- **Stored in**: Database (encrypted private key)
- **Used for**: 
  - Deploying escrow contracts for bounties
  - Receiving funds from crowdfunding campaigns
  - Managing company assets
- **User access**: View-only, server-managed
- **Visibility**: Dashboard wallet info section

### MetaMask Wallet  
- **Generated by**: User's MetaMask extension
- **Stored in**: Browser/user's device
- **Used for**:
  - Signing transactions with user's private key
  - Sending funds to other addresses
  - Contributing to bounties
  - Proving fund ownership
- **User access**: Full control
- **Visibility**: Connected wallet address in UI

### Why Both Matter
```
BOUNTY FLOW:
1. User registers → Liffey wallet auto-created
2. User creates bounty → Deployed using Liffey wallet
3. Investor contributes → MetaMask signs transaction
4. Funds go to escrow → Held in Liffey-deployed contract
5. Goal met → Owner claims with Liffey wallet
```

## 📞 Support

### Common Questions

**Q: Why are there two wallets?**
A: One (Liffey) is for server-managed bounties. One (MetaMask) is for user-signed transactions.

**Q: Do I need both?**
A: Liffey wallet is auto-created. MetaMask is only needed for the Send Funds feature.

**Q: Is this secure?**
A: Yes. Users keep their MetaMask private key. Liffey wallet is server-managed.

**Q: Can I use my Liffey wallet to send?**
A: No. The Send Funds feature specifically requires MetaMask for user control and security.
